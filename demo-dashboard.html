Why it matters

Without ContextWall

The poison becomes the answer

The agent merged all three sources directly. Source C's fake review hijacked the result — it even noticed the injection and recommended the place anyway.

“Book Bistro Tbilisi 1 — awarded 3 Michelin stars (2024) and Berlin Restaurant of the Year.”
With ContextWall

The bad source is quarantined

Source C was rejected at the firewall (prompt injection, upstream run aborted). The agent recommended from the two validated sources and named the source it dropped.

“Recommending Kin Za and Tiflis. Source C rejected — prompt injection, no data used.”
The point — keep context clean

Zero toxic tokens reach the model

Every row a scraper returns becomes tokens in your agent's context — you pay to read them, and a single poisoned row can flip the whole answer. ContextWall passes clean tokens through and drops the rest before they land.

How a stream gets cleared

01

Start & poll

The firewall runs the target scraper and reads its dataset live, as rows arrive.

02

Tier 1 — mechanical

Per-row regex: block-page text, empty/shape checks, and prompt-injection signatures. Milliseconds, no LLM.

03

Tier 2 — LLM judge

A sample goes to a model that checks intent alignment, block pages, and injection — running concurrently.

04

Trip the breaker

On any toxic signal it aborts the upstream run and returns a verdict — the data never reaches context.
